Our Commitment

The SMILE Lab at the University of Florida is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply the relevant accessibility standards to ensure we provide equal access to all users.

Conformance Status

This website aims to conform to the Web Content Accessibility Guidelines (WCAG) 2.2 at Level AAA. WCAG 2.2 became a W3C Recommendation on October 5, 2023 and is also recognized as the ISO/IEC 40500 international standard. Level AAA conformance means we satisfy all Level A, Level AA, and Level AAA success criteria, providing the highest level of accessibility for all users.

About WCAG

The Web Content Accessibility Guidelines are published by the Web Accessibility Initiative (WAI) of the World Wide Web Consortium (W3C). WCAG 2.2 organizes 87 success criteria under four foundational principles, often referred to by the acronym POUR:

1

Perceivable

Information and user interface components must be presentable in ways users can perceive. This includes text alternatives for images, captions for video, sufficient color contrast, and content that can be resized.

2

Operable

User interface components and navigation must be operable by all users. This means keyboard accessibility, enough time to read content, no seizure-inducing flashes, and clear navigation.

3

Understandable

Information and the operation of the interface must be understandable. This includes readable text, predictable behavior, and input assistance such as error identification and labels.

4

Robust

Content must be robust enough to be interpreted by a wide variety of user agents, including assistive technologies. This requires valid markup and proper use of names, roles, and values.

Accessibility Features

This website includes the following accessibility features:

  • Skip navigation links to bypass repeated content and jump directly to main content
  • Semantic HTML with proper landmarks, headings, and ARIA attributes for screen reader compatibility
  • Descriptive alternative text on all meaningful images
  • Keyboard navigable interface with visible focus indicators on all interactive elements
  • Labeled form controls with descriptive error messages
  • Responsive design that supports browser zoom up to 200% without loss of content
  • ARIA live regions that announce dynamic content changes to assistive technologies
  • Enhanced color contrast ratios meeting WCAG AAA requirements (7:1 minimum for body text)
  • Breadcrumb navigation on every sub-page for clear location awareness (WCAG 2.4.8)
  • Abbreviation definitions for acronyms like SMILE, AI, WCAG, NIH, NSF, and BME (WCAG 3.1.4)
  • Enhanced focus indicators with 3px solid outlines for all interactive elements (WCAG 2.4.13)
  • Minimum target size of 44x44 pixels for interactive elements (WCAG 2.5.8)
  • Visual presentation with 1.8x line height and maximum 80-character line width (WCAG 1.4.8)

Known Limitations

Despite our efforts, some content may not yet be fully accessible:

  • PDF documents: Some PDF files, such as the downloadable CV, may not be fully tagged for screen reader use. We are working to improve this.
  • Third-party content: Embedded content from external platforms (e.g., YouTube, Google Maps) relies on those platforms' accessibility implementations.

Compatibility

This website is designed to be compatible with the following assistive technologies:

  • Screen readers (VoiceOver, NVDA, JAWS)
  • Screen magnification software
  • Speech recognition software
  • Modern web browsers (Chrome, Firefox, Safari, Edge)
